# Advanced - ETL 데이터 품질 모니터링

## 떠올린 질문

> ETL이 정상 종료되었다는 사실만 확인하지 말고, 데이터의 유효 비율과
> 오류 유형을 매번 자동으로 기록해서 품질 저하를 발견할 수는 없을까?

## 구현 내용

- 기존 `Total1`의 비동기 수집·검증·저장 함수를 재사용
- 요청·수집·유효·실패 건수와 유효 비율 계산
- 검증 오류를 필드·오류 유형별로 집계
- 카테고리별 건수·평균 가격·총 가격 집계
- 데이터 품질 점수(0~100)와 등급 자동 판정
- JSON, CSV, HTML 형식의 결과물 생성

품질 점수는 `유효 데이터 비율 × 100`을 기본값으로 사용하고, 수집 자체에
실패한 데이터에는 추가 감점을 적용합니다. 계산 기준을 코드에 명시하여
점수의 근거를 확인할 수 있게 했습니다.

## 실행

저장소 최상위 폴더에서 실행합니다.

```powershell
python "과제\Python_data_0720_Day1\Advanced\advanced_data_quality.py"
```

## 생성 결과

- `output/quality_report.html`: 브라우저용 품질 대시보드
- `output/quality_report.json`: 자동 처리 가능한 지표
- `output/validation_failures.csv`: Excel에서 확인 가능한 오류 목록
- `output/etl_output/`: 유효 데이터 CSV·Parquet

## 캡처 권장 화면

1. 터미널의 품질 점수와 생성 파일 경로
2. 브라우저에서 연 `quality_report.html` 전체 화면

## 추가 발전 방향

- 실행 이력을 누적하여 이전 실행 대비 품질 변화 표시
- 품질 점수가 기준 미만이면 이메일 또는 메신저 알림
- 서버 상태에 따라 동시 요청 수를 조절하는 적응형 백프레셔
- 실패 데이터만 다시 처리하는 dead-letter 재처리 기능
